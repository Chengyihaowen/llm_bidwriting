# 08. 测试验收与里程碑

## 1. 验收目标

确保首期系统能稳定完成“上传 -> 解析 -> 编辑 -> 生成 -> 检查 -> 导出”主流程。

## 2. 功能验收清单

## 2.1 招标文件上传与解析

验收项：

- 可上传 PDF、DOCX 文件
- 上传后能看到文件状态
- 解析成功后能生成目录树
- 解析失败后可提示并重试

通过标准：

- 至少一份真实招标文件可完整跑通
- 目录树支持人工继续修改

## 2.2 目录树编辑

验收项：

- 可新增、删除、重命名节点
- 可编辑节点提示词
- 可保存并再次加载

通过标准：

- 页面刷新后目录结构不丢失

## 2.3 分章生成

验收项：

- 可选择一级章节发起生成
- 生成过程流式可见
- 生成结束后正文已保存
- 可重新生成

通过标准：

- 生成任务失败时页面能感知并允许重试

## 2.4 人工编辑

验收项：

- 可直接修改章节内容
- 保存后内容持久化
- 导出时使用修改后的内容

## 2.5 知识库

验收项：

- 可上传知识文件
- 可绑定到项目或章节
- 生成时可启用知识库

通过标准：

- 至少一章内容能明显体现知识库信息参与生成

## 2.6 废标检查

验收项：

- 可手动发起检查
- 可得到结构化问题列表
- 结果可显示风险等级和建议

通过标准：

- 至少能识别出一类高风险项和一类中风险项

## 2.7 Word 导出

验收项：

- 可发起导出任务
- 可下载 Word 文件
- 文件结构与模板一致

通过标准：

- 标题层级、封面、正文顺序均正确

## 3. 非功能测试建议

### 3.1 性能测试

- 大文件上传稳定性
- 章节流式生成首包时间
- 多次保存目录的响应时间

### 3.2 异常测试

- Dify 接口超时
- 文件解析失败
- 流式连接中断
- 导出过程中模板异常

### 3.3 安全测试

- 文件类型校验
- 非法文件名处理
- 敏感配置不下发前端

## 4. 联调顺序建议

建议按以下顺序联调：

1. 项目创建与文件上传
2. 招标文件解析与目录树回显
3. 目录树保存
4. 章节生成流式展示
5. 章节保存与重新生成
6. 知识库绑定
7. 废标检查
8. Word 导出

## 5. 里程碑建议

## 5.1 M1：基础骨架

目标：

- 项目管理
- 文件上传
- 基础页面框架

产出：

- 可创建项目
- 可上传文件

## 5.2 M2：目录解析与编辑

目标：

- 接通 Dify 目录抽取工作流
- 完成树状目录编辑器

产出：

- 自动生成目录树
- 用户可编辑并保存

## 5.3 M3：章节生成

目标：

- 完成章节生成链路
- 完成流式展示与版本保存

产出：

- 可按大章生成
- 可重新生成和手工修改

## 5.4 M4：知识库与废标检查

目标：

- 知识库上传与绑定
- 废标项检查

产出：

- 生成时可引用知识库
- 输出结构化废标风险结果

## 5.5 M5：Word 导出与验收

目标：

- 完成导出模板
- 完成首轮集成验收

产出：

- 可导出 Word
- 主流程可完整跑通

## 6. 主要风险与应对

### 6.1 招标文件质量不稳定

风险：

- PDF 质量差或扫描件导致解析效果差

应对：

- 首期优先支持文本型 PDF / DOCX
- 扫描件作为后续 OCR 增强项

### 6.2 大模型输出不稳定

风险：

- 目录结构不一致、章节内容空泛

应对：

- 结构化输出约束
- 人工可编辑兜底
- 拆工作流减少复杂度

### 6.3 知识库命中率不足

风险：

- 生成结果无法体现企业真实能力

应对：

- 优化知识分类
- 对章节绑定更精细的知识范围

### 6.4 导出格式不符合实际模板

风险：

- 用户仍需大量手工调整

应对：

- 尽早锁定 Word 模板
- 在开发前拿到真实模板样例
